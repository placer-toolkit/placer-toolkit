## Demos

### Appearances

Use the `appearance` attribute to set the button’s appearance.

<CodeDemo>
{`<pc-button appearance="default">Default</pc-button>
<pc-button appearance="primary">Primary</pc-button>
<pc-button appearance="success">Success</pc-button>
<pc-button appearance="warning">Warning</pc-button>
<pc-button appearance="danger">Danger</pc-button>
<pc-button appearance="text">Text</pc-button>`}
</CodeDemo>

### Sizes

Use the `size` attribute to change the button’s size.

<CodeDemo>
{`<pc-button size="small">Small</pc-button>
<pc-button size="medium">Medium</pc-button>
<pc-button size="large">Large</pc-button>`}
</CodeDemo>

### Links

It’s often helpful to have a button that works like a link. This is possible by setting the `href` attribute, which will make the component render as an `<a>` under the hood. This gives you all the default link behaviour the browser provides (e.g., <AutoKeyboardSwitch /> or <kbd>Shift</kbd> + <pc-icon library="default" icon-style="solid" name="arrow-pointer" onmouseover="this.name = 'hand-pointer'" onmouseout="this.name = 'arrow-pointer'" style="cursor: pointer" aria-label="Click"></pc-icon>) and exposes the `target` and `download` attributes.

<CodeDemo>
{`<pc-button href="https://example.com">Link</pc-button>
<pc-button href="https://example.com" target="_blank">New window</pc-button>
<pc-button href="/logo.svg" download>Download</pc-button>
<pc-button href="https://example.com" disabled>Disabled</pc-button>`}
</CodeDemo>

<pc-callout appearance="primary">
    <pc-icon library="default" icon-style="solid" name="circle-info" slot="icon"></pc-icon>
    When a `target` is set, the link will receive `rel="noreferrer noopener"` for [security reasons](https://mathiasbynens.github.io/rel-noopener).
</pc-callout>

### Custom width

As expected, buttons can be given a custom width by passing inline styles to the component (or using a class). This is useful for making buttons span the full width of their container on smaller screens.

<CodeDemo>
{`<pc-button size="small" style="width: 100%; margin-bottom: var(--pc-spacing-m)">Small</pc-button>
<pc-button size="medium" style="width: 100%; margin-bottom: var(--pc-spacing-m)">Medium</pc-button>
<pc-button size="large" style="width: 100%">Large</pc-button>`}
</CodeDemo>

### Custom styles

This example demonstrates how to style buttons using a custom class. This is the recommended approach if you need to add additional appearances. To customise an existing appearance, modify the selector to target the button’s `appearance` attribute instead of a class (e.g., `pc-button[appearance="primary"]`).

<CodeDemo>
{`<pc-button class="custom-pink-button">Pink Button</pc-button>

<style>
    pc-button.custom-pink-button::part(base) {
        background-color: #ff1493;
        border: 2px solid;
        border-top-color: #ff7ac1;
        border-left-color: #ff7ac1;
        border-bottom-color: #ad005c;
        border-right-color: #ad005c;
        border-radius: 6px;
        color: #fff;
        box-shadow: 0 2px 10px #0002;
        transition: all var(--pc-transition-fast) ease-in-out;
    }

    pc-button.custom-pink-button::part(base):hover {
        transform: scale(1.05) rotate(-1deg);
    }

    pc-button.custom-pink-button::part(base):active {
        border-top-color: #ad005c;
        border-right-color: #ff7ac1;
        border-bottom-color: #ff7ac1;
        border-left-color: #ad005c;
        transform: translateY(1px) rotate(-1deg);
    }

    pc-button.custom-pink-button::part(base):focus-visible {
        outline: 2px dashed deeppink;
        outline-offset: 2px;
    }
</style>`}
</CodeDemo>
