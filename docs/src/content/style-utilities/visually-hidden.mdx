This utility makes content accessible to assistive devices without displaying it on the screen.

<blockquote>
    “There are real world situations where visually hiding content may be
    appropriate, while to content should remain available to assistive
    technologies, such as screen readers. For instance, hiding a search field’s
    `label` as a common magnifying glass icon is used in its stead.”
    <footer>
        — <cite>The A11Y Project, How‐to: Hide content</cite>
    </footer>
</blockquote>

Since visually hidden content can receive focus when tabbing, the element will become visible when something inside receives focus. This behaviour is intentional, as otherwise sighted keyboard users wouldn’t be able to determine where the focus indicator is.

```demo
<div style="min-block-size: 1.875rem">
    <a href="#" class="pc-visually-hidden">Skip to main content</a>
</div>
```

## Demos

### Links that open in new windows

In this demo, the link will open in a new window. Screen readers will announce it as “opens in a new window” even though the text content isn’t visible to sighted users.

```demo
<a href="https://example.com" target="_blank" rel="noreferrer noopener">
    Link to example.com
    <pc-icon
        library="default"
        icon-style="solid"
        name="arrow-up-right-from-square"
        style="font-size: 0.75rem; vertical-align: 0.25rem"
        aria-hidden="true"
    ></pc-icon>
    <span class="pc-visually-hidden">opens in a new window</span>
</a>
```

### Content conveyed by context

Adding a label may seem redundant at times, but they’re very helpful for unsighted users. Rather than omit them, you can provide context to unsighted users with visually hidden content that will be announced by assistive devices such as screen readers.

```demo
<pc-card style="inline-size: 100%; max-inline-size: 360px">
    <header class="pc-visually-hidden">
        Personal info
    </header>
    <pc-input label="Name" style="margin-block-end: var(--pc-spacing-m)"></pc-input>
    <pc-input label="E‐mail" type="email"></pc-input>
</pc-card>
```

### Force visually hidden

There are cases where you want to _always_ visually hide certain content, even when it’s focused. For example when hiding a native checkbox to render a custom one:

```demo
<label>
    <span class="checkbox">
        <input type="checkbox" class="pc-visually-hidden-force" />
        <span class="checkbox-tick">
            <pc-icon library="default" icon-style="solid" name="check"></pc-icon>
        </span>
    </span>
    <span class="label">I have read the Terms of Service</span>
</label>

            var(--pc-form-control-border-color);

            in oklab,

            in oklab,
            var(--pc-form-control-activated-color),
            var(--pc-color-mix-active)

    }

    }

```
