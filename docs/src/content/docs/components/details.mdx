Details show a brief summary and expand to show additional content.

```demo:open
<pc-details summary="Toggle me">
    This section contains additional details that are hidden by default. Use details to progressively disclose content that isn’t required immediately, such as explanations, hints or secondary settings.
</pc-details>
```

## Demos

### Variants

Use the `variant` attribute to change the details’ variant.

```demo
<div class="pc-stack">
    <pc-details variant="outlined" summary="Outlined (default)">
        Additional information is hidden until the summary is activated. This variant uses an outlined container to separate it from surrounding content.
    </pc-details>

    <pc-details variant="filled outlined" summary="Filled + Outlined">
        This variant combines a filled background with an outline. Use it when details need stronger visual emphasis.
    </pc-details>

    <pc-details variant="filled" summary="Filled">
        Filled details visually group related information and help distinguish expandable content from the rest of the page.
    </pc-details>

    <pc-details variant="plain" summary="Plain">
        The plain variant removes most visual styling and is best used in dense layouts or inline documentation.
    </pc-details>
</div>
```

### Disabled

Use the `disabled` attribute to prevent the details from expanding.

```demo
<pc-details summary="Disabled" disabled>
    This content cannot be expanded because the Details component is currently disabled. Disabled details remain visible but non‐interactive.
</pc-details>
```

### Customise the summary icon

Use the `expand-icon` and `collapse-icon` slots to change the expand and collapse icons, respectively. To disable the animation, override the `rotate` property on the `summary-icon` part as shown below.

```demo
<pc-details summary="Toggle me" class="custom-icons">
    <pc-icon
        library="default"
        icon-style="solid"
        name="plus"
        slot="expand-icon"
    ></pc-icon>
    <pc-icon
        library="default"
        icon-style="solid"
        name="minus"
        slot="collapse-icon"
    ></pc-icon>

    You can replace the default summary icon using slots. This example uses plus and minus icons and disables the rotation animation.
</pc-details>

<style>
    pc-details.custom-icons::part(summary-icon) {
        rotate: none;
    }
</style>
```

### HTML in summary

To use HTML in the summary, use the `summary` slot. Links and other interactive elements will still retain their behaviour.

```demo
<pc-details>
    <span slot="summary">
        Ooh, look at
        <a href="https://example.com" rel="noreferrer noopener" target="_blank">
            this link</a>!
    </span>

    Interactive elements inside the summary slot retain their default behaviour. Use this sparingly and ensure the summary remains clear and concise.
</pc-details>
```

### RTL language support

The details component automatically adapts to RTL languages with the `lang` and `dir` attributes.

{/* cspell:disable */}

```demo
<pc-details summary="مزيد من المعلومات" lang="ar" dir="rtl">
    يتم استخدام عنصر التفاصيل لعرض محتوى إضافي يمكن إظهاره أو إخفاؤه حسب الحاجة، مع دعم كامل للغات من اليمين إلى اليسار.
</pc-details>
```

### Group details

Details are designed to function independently, but you can simulate a group or “accordion” where only one is shown at a time by listening for the `pc-show` event.

```demo
<div class="details-group pc-stack">
    <pc-details summary="First" open>
        Only one details element in this group can be open at a time. Opening another will automatically close the others.
    </pc-details>

    <pc-details summary="Second">
        This behaviour is implemented by listening to the <code>pc-show</code> event and updating the open state programmatically.
    </pc-details>

    <pc-details summary="Third">
        Grouping details like this is useful for FAQs, settings panels or step‐by‐step explanations.
    </pc-details>
</div>

<script>
    const container = document.querySelector(".details-group");

    container.addEventListener("pc-show", (event) => {
        if (event.target.localName === "pc-details") {
            [...container.querySelectorAll("pc-details")].map((details) => {
                details.open = event.target === details;
            });
        }
    });
</script>
```
